## Gemini CLI Workflow

### Режим A: Headless (auto, stdout) — для рутины
```
Hermes формулирует промпт → Gemini-3.1-pro (headless) → stdout → Hermes сохраняет write_file
```
Когда использовать: новые фичи, багфиксы, рефакторинг

### Режим B: TUI (интерактивный) — для архитектуры
```
Hermes пишет docs/TASK.md → Константин запускает gemini в папке проекта → 
Gemini сам читает, пишет, тестирует → git commit → Hermes деплоит
```
Когда использовать: масштабные изменения, новые системы, когда нужен write_file

### Почему два режима
Google убрал write_file из headless CLI (безопасность). TUI — единственный способ получить полноценного агента-кодера. Headless — быстрее для одноразовых задач.
